## Overview

The LLVM backend provides Just-In-Time (JIT) compilation using LLVM infrastructure. It compiles Yoyo code to native machine code for maximum performance.

<Note>
The LLVM backend is recommended for production workloads and long-running applications where startup time is less critical than peak performance.
</Note>

## LLVMEngine

The execution engine for LLVM-compiled code.

### Class Definition

```cpp include/yoyo/llvm/llvm_engine.h
namespace Yoyo {
    class LLVMEngine : public Engine {
    public:
        llvm::orc::ThreadSafeContext llvm_context;
        std::unique_ptr<llvm::orc::LLJIT> jit;
        
        // Module management
        LLVMAppModule* addAppModule(const std::string& name);
        void addModule(const std::string& module_name, std::string source);
        
        // Compilation
        void compile();
        void prepareForExecution();
        
        // External libraries
        void addStaticLibrary(std::string_view path);
        void addDynamicLibrary(std::string_view path);
        
        // Constants
        virtual void* createGlobalConstant(
            const Type& type,
            const std::vector<Constant>& args,
            IRGenerator*
        ) override;
    };
}
```

## LLVMAppModule

Extends `LLModule` with FFI capabilities for binding C++ functions.

```cpp include/yoyo/llvm/app_module.h
namespace Yoyo {
    class LLVMAppModule : public LLModule {
    public:
        template<typename R, typename... Args>
        Result addFunction(
            std::string sig_str,
            R(*func)(Args...),
            std::string name
        );
        
        Result addEnum(
            std::string enum_name,
            std::unordered_map<std::string, int32_t> values
        );
        
        Result addClass(
            std::string class_name,
            std::vector<ClassVariable> variables,
            Ownership sh,
            std::string block_hash
        );
    };
}
```

### addFunction

```cpp
template<typename R, typename... Args>
Result addFunction(
    std::string sig_str,
    R(*func)(Args...),
    std::string name
);
```

Bind a C++ function to make it callable from Yoyo code.

<ParamField path="sig_str" type="std::string">
Yoyo function signature string (e.g., `"fn add(x: i32, y: i32) -> i32"`).
</ParamField>

<ParamField path="func" type="R(*)(Args...)">
C++ function pointer.
</ParamField>

<ParamField path="name" type="std::string">
Name to expose in Yoyo.
</ParamField>

**Returns:** `Result` indicating success or specific error.

**Example:**
```cpp
int32_t add(int32_t a, int32_t b) {
    return a + b;
}

LLVMEngine engine;
auto* app = engine.addAppModule("math");

Result res = app->addFunction(
    "fn add(x: i32, y: i32) -> i32",
    &add,
    "add"
);

if (res == Result::Success) {
    // Function is now callable from Yoyo
}
```

## Usage Example

```cpp
#include <yoyo/llvm/llvm_engine.h>

using namespace Yoyo;

int main() {
    // Create engine
    LLVMEngine engine;
    
    // Add Yoyo source code
    std::string source = R"(
        fn fibonacci(n: i32) -> i32 {
            if n <= 1 {
                return n;
            }
            return fibonacci(n - 1) + fibonacci(n - 2);
        }
    )";
    
    engine.addModule("main", source);
    
    // Compile to native code
    engine.compile();
    engine.prepareForExecution();
    
    // Find and call the function
    auto* fib = engine.findFunction("fibonacci");
    int32_t result = reinterpret_cast<int32_t(*)(int32_t)>(fib)(10);
    
    std::cout << "fibonacci(10) = " << result << std::endl;
    
    return 0;
}
```
