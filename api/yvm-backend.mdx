## Overview

The Yoyo Virtual Machine (YVM) backend compiles Yoyo code to bytecode for interpretation. It offers significantly faster compilation times than LLVM at the cost of some runtime performance.

<Note>
The YVM backend is ideal for development, scripting, and applications where fast startup time matters more than peak execution speed.
</Note>

### Stack-Based Evaluation

Unlike LLVM which uses SSA form, YVM uses a stack-based architecture:

- Expressions push results onto the stack
- `operator()` methods return `std::vector<Type>` describing what was pushed
- `on_stack` parameter controls whether to use pre-allocated stack space
- `perform_load` controls whether to output values or just addresses

## Usage Example

```cpp
#include <yoyo/yvm/yvm_engine.h>

using namespace Yoyo;

int main() {
    // Create engine
    YVMEngine engine;
    
    // Add Yoyo source code
    std::string source = R"(
        fn factorial(n: i32) -> i32 {
            if n <= 1 {
                return 1;
            }
            return n * factorial(n - 1);
        }
    )";
    
    auto* mod = engine.addModule("main", source);
    
    // Compile to bytecode
    if (!engine.compile()) {
        std::cerr << "Compilation failed" << std::endl;
        return 1;
    }
    
    engine.prepareForExecution();
    
    // Find the function
    auto fn_opt = engine.findFunction(mod, "factorial");
    if (!fn_opt) {
        std::cerr << "Function not found" << std::endl;
        return 1;
    }
    
    // Create a fiber and execute
    Fiber fiber = engine.createFiber(fn_opt.value());
    
    // Call with argument 5
    int32_t arg = 5;
    int32_t result;
    fiber.call(&arg, &result);
    
    std::cout << "factorial(5) = " << result << std::endl;
    
    return 0;
}
```

## LLVM vs YVM Comparison

<Tabs>
  <Tab title="Compilation Speed">
    **YVM**: 10-100x faster compilation
    
    YVM generates bytecode directly from the AST, while LLVM performs multiple optimization passes.
  </Tab>
  <Tab title="Runtime Performance">
    **LLVM**: 2-5x faster execution
    
    LLVM produces optimized native code, while YVM interprets bytecode.
  </Tab>
  <Tab title="Startup Time">
    **YVM**: Near-instant
    
    Perfect for scripting and development workflows.
  </Tab>
  <Tab title="Memory Usage">
    **YVM**: ~10MB overhead
    
    **LLVM**: ~50-100MB overhead
    
    YVM is more suitable for embedded scenarios.
  </Tab>
  <Tab title="Debugging">
    **YVM**: Easier to debug with bytecode dumps
    
    **LLVM**: More complex but supports standard native debuggers
  </Tab>
</Tabs>
