The `ModuleBase` class provides the foundation for Yoyo's module system, managing functions, classes, types, and other declarations within a module scope.

## Class Definition

```cpp
class ModuleBase
{
public:
    struct FunctionDetails {
        std::string name;
        FunctionSignature sig;
        std::vector<Attribute> attributes;
        bool is_private() const;
    };
    
    virtual ~ModuleBase() = default;
    
    // Module hierarchy
    Engine* engine;
    std::string module_hash;
    std::unordered_map<std::string, ModuleBase*> modules;
    
    // ... declaration maps
};
```

## FunctionDetails

Nested structure containing metadata about module functions.

```cpp
struct FunctionDetails
{
    std::string name;
    FunctionSignature sig;
    std::vector<Attribute> attributes;
    bool is_private() const;
};
```

### Members

<ParamField path="name" type="std::string">
  The function name
</ParamField>

<ParamField path="sig" type="FunctionSignature">
  The function's signature including parameter types and return type
</ParamField>

<ParamField path="attributes" type="std::vector<Attribute>">
  Function attributes (e.g., `@public`, `@inline`, etc.)
</ParamField>

### is_private()

Determines if a function is private (not marked with `@public` attribute).

```cpp
bool is_private() const;
```

#### Returns

<ResponseField name="return" type="bool">
  `true` if the function is private (no `@public` attribute), `false` if public
</ResponseField>

#### Example

```cpp
Yoyo::ModuleBase::FunctionDetails details;
details.name = "helper";
// No @public attribute added

if (details.is_private()) {
    std::cout << "Function is private to the module" << std::endl;
}
```

## Lookup Methods

The module provides various lookup methods for finding declarations within a module hierarchy.

### findFunction()

Finds a function by name within a specific block scope.

```cpp
std::pair<std::string, FunctionDetails*> findFunction(
    const std::string& block, 
    const std::string& name
);
```

#### Parameters

<ParamField path="block" type="const std::string&" required>
  The block/scope name to search in
</ParamField>

<ParamField path="name" type="const std::string&" required>
  The function name to find
</ParamField>

#### Returns

<ResponseField name="return" type="std::pair<std::string, FunctionDetails*>">
  Pair of (qualified name, pointer to function details), or (empty, nullptr) if not found
</ResponseField>

#### Example

```cpp
auto [qualifiedName, funcDetails] = module.findFunction("MyClass", "myMethod");
if (funcDetails != nullptr) {
    std::cout << "Found function: " << funcDetails->name << std::endl;
}
```

## Usage Example

```cpp
#include <yoyo/module.h>
#include <yoyo/engine.h>
#include <iostream>

class MyModule : public Yoyo::ModuleBase {
public:
    MyModule(Yoyo::Engine* eng) {
        engine = eng;
        module_hash = "my_module_v1";
    }
    
    void registerFunction(const std::string& name, 
                         const Yoyo::FunctionSignature& sig) {
        FunctionDetails details;
        details.name = name;
        details.sig = sig;
        functions[name].push_back(details);
    }
};

int main() {
    MyEngine engine;
    MyModule module(&engine);
    
    // Register the module
    engine.modules["MyModule"] = std::make_unique<MyModule>(module);
    
    // Look up a function
    auto [qualName, func] = module.findFunction("", "myFunction");
    if (func) {
        std::cout << "Found function: " << func->name << std::endl;
        if (func->is_private()) {
            std::cout << "Function is private" << std::endl;
        }
    }
    
    return 0;
}
```

<Note>
The ModuleBase class is designed to be extended. You typically don't instantiate it directly, but create derived classes that implement module-specific functionality.
</Note>
