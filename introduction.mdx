# Welcome to Yoyo

Yoyo is a statically typed, memory-safe scripting language designed to be embedded in C++20 applications. With JIT compilation support through LLVM and a custom VM backend, Yoyo provides high performance while maintaining memory safety guarantees through compile-time borrow checking.

<Note>
This implementation is currently in active development and is very far from production-ready use.
</Note>

## Why Yoyo?

Yoyo brings together modern language features with the safety and performance requirements of systems programming:

- **Memory Safety**: Borrow checker prevents use-after-free, data races, and memory leaks at compile time
- **Zero-Cost Abstractions**: Generics and interfaces compile down to efficient machine code
- **C++20 Integration**: Native interoperability with modern C++ codebases
- **Flexible Execution**: Choose LLVM for maximum performance or YVM for faster compilation times

## Syntax Overview

Yoyo's syntax is heavily inspired by [cpp2/cppfront](https://github.com/hsutter/cppfront) with all declarations following the form:

```yoyo
name: kind = value;
```

Here's a quick example:

```yoyo
// Variable declarations (type can be inferred)
x: mut u32 = 42;

// Function declarations
add: fn (a: u32, b: u32) -> u32 = return a + b;

// Implicit block syntax for single-statement functions
square: fn (x: u32) -> u32 = return x * x;

// Struct definitions
Point: struct = {
    x: f64,
    y: f64
}
```
