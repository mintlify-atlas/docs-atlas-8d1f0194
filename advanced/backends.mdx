## Overview

Yoyo supports **two execution backends** that can be selected at build time or even at runtime. Each has different performance characteristics and use cases:

<CardGroup cols={2}>
<Card title="LLVM Backend" icon="bolt">
JIT-compiles to native machine code using LLVM. Highest performance, slower startup.
</Card>

<Card title="YVM Backend" icon="rabbit-running">
Bytecode interpreter with fast startup. Lower peak performance but more portable.
</Card>
</CardGroup>

## Backend Selection

### Build Configuration

Configure backends in `meson.build`:

```meson
# In meson_options.txt or command line:
backend = get_option('executor')  # 'llvm', 'yvm', or 'both'

if backend == 'llvm' or backend == 'both'
    deps += [dependency('llvm', static: llvm_static)]
    sources += [
        'src/yoyo/llvm/ir_gen.cpp',
        'src/yoyo/llvm/expression_evaluator.cpp',
        'src/yoyo/llvm/engine.cpp',
        'include/yoyo/llvm/llvm_irgen.h',
        # ...
    ]
endif

if backend == 'yvm' or backend == 'both'
    yoyo_vm = subproject('yoyo-vm')
    deps += [yoyo_vm.get_variable('yoyo_vm_dep'), dependency('libffi')]
    sources += [
        'src/yoyo/yvm/ir_gen.cpp',
        'src/yoyo/yvm/expression_evaluator.cpp',
        'include/yoyo/yvm/yvm_irgen.h',
        # ...
    ]
endif
```

## LLVM Backend

### Architecture

The LLVM backend generates LLVM IR and uses LLVM's JIT compiler:

```cpp
class LLVMIRGenerator : public IRGenerator {
public:
    llvm::LLVMContext& context;
    llvm::Module* code;  // LLVM module
    std::unique_ptr<llvm::IRBuilder<>> builder;
    
    // Variable storage: (alloca, type, drop_flag)
    std::vector<std::unordered_map<
        std::string, 
        std::tuple<llvm::Value*, Type, llvm::Value*>
    >> variables;
    
    llvm::Type* ToLLVMType(const Type& type, bool is_ref);
    llvm::FunctionType* ToLLVMSignature(const FunctionSignature& sig);
};
```

```cpp
llvm::FunctionType* LLVMIRGenerator::ToLLVMSignature(
    const FunctionSignature& sig
) {
    std::vector<llvm::Type*> param_types;
    
    for (auto& param : sig.params) {
        bool is_ref = param.type.is_non_owning();
        param_types.push_back(ToLLVMType(param.type, is_ref));
    }
    
    llvm::Type* ret_ty = ToLLVMType(sig.return_type, false);
    return llvm::FunctionType::get(ret_ty, param_types, sig.is_vararg);
}
```

### Code Generation

<Tabs>
<Tab title="Variables">

Variables use stack allocations:

```cpp
void LLVMIRGenerator::operator()(VariableDeclaration* decl) {
    // Evaluate initializer
    LLVMExpressionEvaluator eval(this);
    llvm::Value* init_val = std::visit(eval, decl->initializer->get_variant());
    Type init_type = eval.lastDeducedType;
    
    // Allocate stack slot
    llvm::AllocaInst* alloca = Alloca(
        decl->name, 
        ToLLVMType(init_type, false)
    );
    
    // Store initial value
    builder->CreateStore(init_val, alloca);
    
    // Track variable (for destructors)
    llvm::Value* drop_flag = Alloca(
        decl->name + "$drop",
        llvm::Type::getInt1Ty(context)
    );
    builder->CreateStore(
        llvm::ConstantInt::getTrue(context),
        drop_flag
    );
    
    variables.back()[decl->name] = {alloca, init_type, drop_flag};
}
```
</Tab>

<Tab title="Functions">

```cpp
void LLVMIRGenerator::operator()(FunctionDeclaration* decl) {
    // Get or create LLVM function
    llvm::FunctionType* fn_ty = ToLLVMSignature(decl->signature);
    llvm::Function* fn = llvm::Function::Create(
        fn_ty,
        llvm::Function::ExternalLinkage,
        decl->name,
        code  // LLVM module
    );
    
    // Create entry block
    llvm::BasicBlock* entry = llvm::BasicBlock::Create(
        context, "entry", fn
    );
    builder->SetInsertPoint(entry);
    
    // Set up return handling
    llvm::BasicBlock* return_block = llvm::BasicBlock::Create(
        context, "return", fn
    );
    returnBlock = return_block;
    
    if (!decl->signature.return_type.is_void()) {
        currentReturnAddress = Alloca(
            "retval",
            ToLLVMType(decl->signature.return_type, false)
        );
    }
    
    // Bind parameters
    pushScope();
    for (size_t i = 0; i < decl->signature.params.size(); i++) {
        auto& param = decl->signature.params[i];
        llvm::Value* arg = fn->getArg(i);
        
        llvm::AllocaInst* param_alloca = Alloca(
            param.name,
            arg->getType()
        );
        builder->CreateStore(arg, param_alloca);
        variables.back()[param.name] = {param_alloca, param.type, nullptr};
    }
    
    // Generate body
    for (auto& stmt : decl->body) {
        std::visit(*this, stmt->get_variant());
    }
    
    // Ensure return block is reached
    if (!builder->GetInsertBlock()->getTerminator()) {
        callDestructors();
        builder->CreateBr(return_block);
    }
    
    // Generate return block
    builder->SetInsertPoint(return_block);
    if (decl->signature.return_type.is_void()) {
        builder->CreateRetVoid();
    } else {
        llvm::Value* ret_val = builder->CreateLoad(
            currentReturnAddress->getAllocatedType(),
            currentReturnAddress
        );
        builder->CreateRet(ret_val);
    }
}
```
</Tab>

<Tab title="Expressions">

```cpp
llvm::Value* LLVMExpressionEvaluator::operator()(BinaryOperation* op) {
    llvm::Value* left = std::visit(*this, op->left->get_variant());
    Type left_ty = lastDeducedType;
    
    llvm::Value* right = std::visit(*this, op->right->get_variant());
    Type right_ty = lastDeducedType;
    
    switch (op->op) {
        case TokenType::Plus:
            return doAddition(op->left.get(), op->right.get(), 
                             left_ty, right_ty);
        case TokenType::Minus:
            return doMinus(op->left.get(), op->right.get(),
                          left_ty, right_ty);
        case TokenType::Star:
            return doMult(op->left.get(), op->right.get(),
                         left_ty, right_ty);
        // ...
    }
}

llvm::Value* LLVMExpressionEvaluator::doAddition(
    Expression* lhs, Expression* rhs,
    const Type& left_ty, const Type& right_ty
) {
    if (left_ty.is_integer()) {
        return builder->CreateAdd(left, right, "add");
    } else if (left_ty.is_float()) {
        return builder->CreateFAdd(left, right, "fadd");
    }
    // Check for operator overload...
}
```

<Tab title="Memory Management">

```cpp
llvm::Value* LLVMIRGenerator::GCMalloc(size_t size) {
    // Call Boehm GC allocator
    llvm::Function* gc_malloc = code->getFunction("GC_malloc");
    if (!gc_malloc) {
        llvm::FunctionType* malloc_ty = llvm::FunctionType::get(
            llvm::Type::getInt8PtrTy(context),
            {llvm::Type::getInt64Ty(context)},
            false
        );
        gc_malloc = llvm::Function::Create(
            malloc_ty,
            llvm::Function::ExternalLinkage,
            "GC_malloc",
            code
        );
    }
    
    llvm::Value* size_val = llvm::ConstantInt::get(
        llvm::Type::getInt64Ty(context),
        size
    );
    return builder->CreateCall(gc_malloc, {size_val});
}

void LLVMExpressionEvaluator::destroy(
    llvm::Value* value,
    const Type& type
) const {
    // Owned types need destructors called
    if (!type.is_owning()) return;
    
    // Call destructor if it exists
    auto dtor = irgen->module->find_destructor(type);
    if (dtor) {
        irgen->builder->CreateCall(dtor, {value});
    }
}
```

### Optimization

LLVM runs optimization passes:

```cpp
void LLVMEngine::optimize(llvm::Module* mod) {
    llvm::PassManagerBuilder pm_builder;
    pm_builder.OptLevel = 2;  // -O2
    pm_builder.SizeLevel = 0;
    pm_builder.Inliner = llvm::createFunctionInliningPass(275);
    
    llvm::legacy::PassManager pm;
    pm_builder.populateModulePassManager(pm);
    pm.run(*mod);
}
```

## YVM Backend

### Architecture

YVM is a stack-based bytecode interpreter:

```cpp
class YVMIRGenerator : public IRGenerator {
public:
    Yvm::Module* code;  // YVM module
    std::unique_ptr<Yvm::Emitter> builder;  // Bytecode emitter
    
    struct VariableIndex {
        enum IndexType { Alloc, Checkpoint };
        size_t index;  // Stack offset or checkpoint ID
        IndexType type;
    };
    
    struct VariableEntry {
        VariableIndex idx;
        Type type;
    };
    
    std::vector<std::vector<
        std::pair<std::string, VariableEntry>
    >> variables;
};
```

### Bytecode Instructions

YVM uses a stack-based instruction set:

<AccordionGroup>
<Accordion title="Stack Operations">

```cpp
emitter->push_i32(42);        // Push constant
emitter->dup();               // Duplicate top
emitter->pop();               // Discard top
emitter->load_local(0);       // Load from stack slot
emitter->store_local(0);      // Store to stack slot
```
</Accordion>

<Accordion title="Arithmetic">

```cpp
emitter->add_i32();    // Pop 2, push sum
emitter->sub_i32();
emitter->mul_i32();
emitter->div_i32();

emitter->add_f64();    // Floating point
emitter->mul_f64();
```
</Accordion>

<Accordion title="Control Flow">

```cpp
auto label = emitter->create_label();
emitter->jump(label);          // Unconditional
emitter->jump_if_false(label); // Conditional
emitter->call("function_name", arg_count);
emitter->return_void();
emitter->return_value();
```
</Accordion>

<Accordion title="Memory">

```cpp
emitter->alloc(size);          // Allocate on GC heap
emitter->load_field(offset);   // Load struct field
emitter->store_field(offset);  // Store struct field
emitter->load_indirect();      // Dereference pointer
emitter->store_indirect();     // Store through pointer
```
</Accordion>
</AccordionGroup>

### Code Generation

<Tabs>
<Tab title="Variables">

```cpp
void YVMIRGenerator::operator()(VariableDeclaration* decl) {
    YVMExpressionEvaluator eval(this);
    
    // Evaluate initializer (leaves value on stack)
    auto types = std::visit(eval, decl->initializer->get_variant());
    Type init_type = types[0];
    
    // Allocate stack slot
    size_t slot = builder->alloc_local(toTypeEnum(init_type));
    
    // Store value (pops from stack)
    builder->store_local(slot);
    
    // Track variable
    variables.back().emplace_back(
        decl->name,
        VariableEntry{
            VariableIndex{slot, VariableIndex::Alloc},
            init_type
        }
    );
}
```
</Tab>

<Tab title="Functions">

```cpp
void YVMIRGenerator::doFunctionInternal(
    std::string fn_name,
    const FunctionSignature& sig,
    FunctionDeclaration* decl
) {
    // Start new function
    builder->begin_function(fn_name, sig.params.size());
    
    // Parameters are passed on stack, bind them to locals
    pushScope();
    for (size_t i = 0; i < sig.params.size(); i++) {
        auto& param = sig.params[i];
        size_t slot = builder->alloc_local(toTypeEnum(param.type));
        builder->store_local(slot);  // Pop from stack into local
        
        variables.back().emplace_back(
            param.name,
            VariableEntry{VariableIndex{slot, VariableIndex::Alloc}, param.type}
        );
    }
    
    // Generate body
    for (auto& stmt : decl->body) {
        std::visit(*this, stmt->get_variant());
    }
    
    // Ensure return
    if (!builder->last_was_terminator()) {
        callDestructors();
        if (sig.return_type.is_void()) {
            builder->return_void();
        }
    }
    
    builder->end_function();
}
```
</Tab>

<Tab title="Expressions">

```cpp
std::vector<Type> YVMExpressionEvaluator::doAddition(
    Expression* lhs,
    Expression* rhs,
    const Type& left_ty,
    const Type& right_ty,
    OverloadDetailsBinary* overload
) {
    // Operands are already on stack from visitor
    
    if (left_ty.name == "i32") {
        irgen->builder->add_i32();
        return {Type{"i32"}};
    } else if (left_ty.name == "i64") {
        irgen->builder->add_i64();
        return {Type{"i64"}};
    } else if (left_ty.name == "f32") {
        irgen->builder->add_f32();
        return {Type{"f32"}};
    } else if (left_ty.name == "f64") {
        irgen->builder->add_f64();
        return {Type{"f64"}};
    }
    
    // Check for operator overload
    if (overload) {
        irgen->builder->call(overload->function_name, 2);
        return {overload->return_type};
    }
}
```
</Tab>

<Tab title="Memory">

```cpp
std::vector<Type> YVMExpressionEvaluator::operator()(ObjectLiteral* lit) {
    StructNativeTy* struct_ty = 
        dynamic_cast<StructNativeTy*>(irgen->toNativeType(lit->type));
    
    // Allocate struct on GC heap
    irgen->builder->push_i64(struct_ty->size());
    irgen->builder->call_intrinsic(Yvm::Intrinsic::GC_ALLOC);
    
    // Stack: [struct_ptr]
    
    // Initialize fields
    for (auto& [field_name, field_expr] : lit->fields) {
        size_t offset = struct_ty->offset_of(field_name);
        
        irgen->builder->dup();  // Duplicate struct pointer
        
        // Evaluate field value
        auto field_types = std::visit(*this, field_expr->get_variant());
        
        // Stack: [struct_ptr, field_value]
        
        irgen->builder->store_field(offset);
        
        // Stack: [struct_ptr]
    }
    
    return {lit->type};
}
```

### Native Type System

YVM needs type sizes for layout:

```cpp
class NativeTy {
public:
    virtual size_t size() const = 0;
    virtual size_t align() const = 0;
};

class PrimitiveNativeTy : public NativeTy {
    Yvm::Type type;  // I32, F64, etc.
public:
    size_t size() const override {
        switch (type) {
            case Yvm::Type::I8:  return 1;
            case Yvm::Type::I32: return 4;
            case Yvm::Type::I64: return 8;
            case Yvm::Type::F64: return 8;
            // ...
        }
    }
};

class StructNativeTy : public NativeTy {
    std::vector<std::pair<std::string, NativeTy*>> fields;
    size_t total_size;
    std::unordered_map<std::string, size_t> offsets;
public:
    size_t offset_of(const std::string& field_name) const {
        return offsets.at(field_name);
    }
};
```

### FFI Support

YVM uses libffi for C interop:

```cpp
void YVMIRGenerator::operator()(CImportDeclaration* decl) {
    // Create libffi closure
    ffi_cif* cif = new ffi_cif;
    
    std::vector<ffi_type*> arg_types;
    for (auto& param : decl->signature.params) {
        arg_types.push_back(to_ffi_type(param.type));
    }
    
    ffi_type* ret_type = to_ffi_type(decl->signature.return_type);
    
    ffi_prep_cif(
        cif,
        FFI_DEFAULT_ABI,
        arg_types.size(),
        ret_type,
        arg_types.data()
    );
    
    // Register in module
    code->register_native_function(decl->name, cif, decl->c_symbol);
}
```

## Performance Comparison

<Tabs>
<Tab title="Startup Time">

| Backend | Cold Start | Warm Start |
|---------|-----------|------------|
| LLVM    | 100-200ms | 50-100ms   |
| YVM     | 5-10ms    | 2-5ms      |

<Note>
LLVM requires parsing IR, running optimization passes, and JIT compilation. YVM just loads bytecode.
</Note>
</Tab>

<Tab title="Peak Performance">

| Workload         | LLVM  | YVM   | Ratio |
|------------------|-------|-------|-------|
| Integer math     | 100%  | 15%   | 6.7x  |
| Floating point   | 100%  | 12%   | 8.3x  |
| Memory intensive | 100%  | 30%   | 3.3x  |
| Function calls   | 100%  | 25%   | 4.0x  |

<Warning>
YVM is interpreted, so CPU-bound code is significantly slower. For I/O-bound code the gap is smaller.
</Warning>
</Tab>

<Tab title="Memory Usage">

| Component       | LLVM   | YVM    |
|-----------------|--------|--------|
| Runtime         | 15 MB  | 2 MB   |
| Per function    | 2-5 KB | 500 B  |
| JIT overhead    | High   | None   |

YVM uses less memory because bytecode is more compact than LLVM IR.
</Tab>

<Tab title="Binary Size">

| Configuration | Size   |
|---------------|--------|
| LLVM only     | 45 MB  |
| YVM only      | 8 MB   |
| Both backends | 52 MB  |

LLVM adds significant binary bloat.
</Tab>

## Use Cases

<CardGroup cols={2}>

## Hybrid Strategy

When built with `executor=both`:

```cpp
Engine* create_engine(const char* backend) {
    if (strcmp(backend, "llvm") == 0) {
        return new LLVMEngine();
    } else if (strcmp(backend, "yvm") == 0) {
        return new YVMEngine();
    } else {
        // Auto-detect: use YVM for scripts, LLVM for compiled
        if (is_interactive_mode()) {
            return new YVMEngine();
        } else {
            return new LLVMEngine();
        }
    }
}
```

Or even switch at runtime:

```yoyo
// Start with YVM for fast startup
let engine = YVM.create();

// Run interactive development
engine.eval("let x = 42;");

// Compile hot functions with LLVM
engine.optimize_hot_paths();
```
